# CrewClaw

> 🧠 面向团队的 OpenClaw 工作区控制平面

CrewClaw 是一个开源平台，帮助团队**按用户安全地开通、管理与扩展 OpenClaw 工作区**，并在控制平面、运行时编排与运行时环境之间保持清晰边界。

为团队而生——而非仅限个人使用。

---

## ✨ 为什么选择 CrewClaw？

OpenClaw 能力强大，但面向**团队**落地时会遇到现实问题：

* 如何管理**多用户与多工作区**？
* 如何按用户隔离环境？
* 如何实现**邀请流程、角色与权限**？
* 如何安全暴露运行时 UI？
* 如何扩展运行时的生命周期（启停、恢复）？

CrewClaw 针对上述问题提供一体化方案。

---

## 🧩 架构概览

CrewClaw 采用**边界优先**的架构：浏览器入口、身份、控制平面、运行时编排、每用户运行时，以及统一模型层彼此分离。

高层数据流如下：

```text
Browser
  |
  v
Traefik -> Authentik -> CrewClaw 控制平面 -> 运行时编排器 -> Runtime Manager -> 每用户 OpenClaw 运行时
                                      |
                                      +-> 模型服务 / 凭据代理 -> LiteLLM -> vLLM / Ollama / 上游提供商
                                      |
                                      +-> PostgreSQL
```

### 分层说明

| 层级 | 组件 | 职责 |
| ---- | ---- | ---- |
| 入口层 | Traefik + Authentik | 路由、登录、会话校验、子域名保护 |
| 平台层 | CrewClaw 控制平面 + Web UI | 用户同步、运行时真相、管理治理、工作区入口 |
| 编排层 | 运行时编排器 + runtime manager | 期望状态对齐、配置渲染、容器生命周期 |
| 运行时层 | 每用户 OpenClaw 运行时 | 用户工作区、配置、智能体运行时 |
| 模型层 | LiteLLM + PostgreSQL + vLLM/Ollama | 统一模型访问、凭据代理、用量汇总 |

**MVP 设计要点**：每用户一个运行时；`browserUrl`（面向浏览器）与 `internalEndpoint`（平台/内部）字段分离、不可混用；工作区子域名仍须经 Traefik/Authentik 鉴权；控制平面不直接操作 Docker——容器生命周期由 runtime manager 负责。

完整说明见 [ARCHITECTURE.md](ARCHITECTURE.md)（英文）。

### 经典三段式理解

### 1. 控制平面（CrewClaw）

* 用户 / 工作区 / 角色管理
* 邀请体系
* 运行时生命周期作为**业务真相**
* 与 Authentik（身份）集成
* 管理端与用户端控制台

### 2. 运行时编排（RuntimeManager）

* 创建 / 启动 / 停止 / 删除运行时
* 容器生命周期管理
* 健康检查与状态上报
* 无状态执行层

### 3. 运行时环境（OpenClaw Docker）

* 每用户隔离的运行时
* 挂载工作区
* 浏览器可访问的 UI

```
用户 → CrewClaw → RuntimeManager → 运行时（OpenClaw）
```

## 🚀 主要特性

### 🏢 团队优先

* 多用户、多工作区
* 基于角色的访问控制
* 以邀请驱动的入驻流程

### 🔐 默认安全

* Authentik 集成
* Traefik + Outpost 路由
* 每运行时隔离

### ⚙️ 运行时生命周期

* 声明式运行时状态
* 自动恢复与状态同步
* 保留策略（retention）

### 🌐 浏览器访问

* 每个运行时暴露安全 UI
* 外网与内网端点分离

### 🧱 清晰架构

* 控制平面 ≠ 运行时 ≠ 编排层
* 各层解耦

## 📦 代码仓库

本项目由多个仓库组成：

### 🧠 crewclaw（本仓库）

> 控制平面

* API 服务
* 管理端 / 用户端控制台
* 业务逻辑（用户、工作区、邀请）
* 运行时生命周期真相源

---

### ⚙️ runtimemanager

> 运行时编排引擎

* 负责运行时创建与全生命周期
* 向 CrewClaw 回传运行时状态
* 可独立运行或通过 Docker 部署

---

### 🐳 openclaw-docker

> 参考运行时镜像

* 标准 OpenClaw 运行时环境
* 预配置以对接 CrewClaw

## 💡 设计原则

* **优先分离关注点**
* **控制平面持有真相**
* **运行时可丢弃、可重建**
* **编排层无状态**
* **默认安全**
